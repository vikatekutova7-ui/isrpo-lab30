## Таблица атрибутов JSON
| Атрибут / настройка | Что делает | Пример |
|---------------------|------------|--------|
| `[JsonPropertyName("name")]` | Задаёт имя поля в JSON | `"name"` вместо `"Name"` |
| `[JsonIgnore]` | Поле не попадает в JSON | Пароли, внутренние заметки |
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase | `"realName"` вместо `"RealName"` |
| `JsonStringEnumConverter` | Enum как строка | `"Marvel"` вместо `0` |
| `WriteIndented = true` | JSON с отступами | Для удобства при разработке |

## Главные выводы
### 1. Что такое сериализация и десериализация?
**Сериализация** — это процесс преобразования C#-объекта в JSON-строку для передачи по сети.  
**Десериализация** — обратный процесс: из JSON-строки создаётся C#-объект.
### 2. Зачем нужен `[JsonIgnore]`?
`[JsonIgnore]` исключает свойство из JSON-ответа. Это нужно для:
- **Скрытия чувствительных данных** — например, пароля пользователя, токена доступа
- **Внутренних служебных полей** — заметки разработчика (`InternalNotes`), отладочная информация
- **Свойств, которые не должны попадать на клиент** — например, `LastLoginIp` или `HashedPassword`
Пример в коде:
```
[JsonIgnore]
public string? InternalNotes { get; set; } // Не попадёт в JSON
```
### 3. Почему enum лучше сериализовывать строкой, а не числом?
| Способ | Пример | Проблема |
|---|---|---|
| Число (по умолчанию) | `"universe": 0` | Непонятно, что означает `0` — Marvel? DC? Ошибка? |
| Строка (с `JsonStringEnumConverter`) | `"universe": "Marvel"` | Человекопонятно, удобно при отладке, клиент видит осмысленное значение |

**Вывод:** Строковая сериализация делает API самодокументируемым и снижает вероятность ошибок при интеграции.
### 4. Что изменилось в ответе после настройки CamelCase?

**До настройки (`PropertyNamingPolicy = null` или по умолчанию):**
```
{
  "Id": 1,
  "Name": "Человек-паук",
  "RealName": "Питер Паркер",
  "PowerLevel": 75
}
```

**После настройки (`CamelCase`):**
```
{
  "id": 1,
  "name": "Человек-паук",
  "realName": "Питер Паркер",
  "powerLevel": 75
}
``` 
Все имена свойств стали начинаться с маленькой буквы (camelCase), что соответствует стандартам JavaScript/JSON API и ожиданиям фронтенд-разработчиков.
